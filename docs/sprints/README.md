# Sprints ativas MIDWAY

Esta pasta guarda o planejamento vivo de evolução do MIDWAY.

## Regra de separação

- `docs/sprints/` contém sprints ativas ou planejadas.
- `docs/historico/sprint/` contém sprints antigas, apenas como memória técnica.
- Nenhuma sprint histórica deve ser tratada como especificação vigente sem migração explícita para `docs/` ou `docs/modulos/`.

## Regra de transição segura

Toda sprint deve declarar:

1. objetivo;
2. escopo;
3. fora de escopo;
4. documentos de referência ativos;
5. riscos de regressão;
6. plano de validação;
7. critério de aceite;
8. plano de rollback ou contenção;
9. impacto em React, Streamlit, API, banco e exportação IQS.
